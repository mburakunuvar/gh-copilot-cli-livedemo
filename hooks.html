Adding Hooks for GitHub Copilot CLI

Hooks let you run custom shell commands at key points in Copilot agent sessions so you can automate logging, notifications, validation, and other workflow steps.

Prerequisite (Windows)

Install PowerShell 7+ and confirm it is in your PATH:

pwsh --version (install with winget install Microsoft.PowerShell if needed).

Create a repository-level hook

  1. Create .github\hooks\NAME.json.
  2. Use a top-level structure with "version": 1 and a "hooks" object.
  3. Configure commands per trigger (for example: sessionStart, preToolUse, postToolUse, errorOccurred).
  4. Include both bash and powershell keys for cross-platform support.
  5. Commit and merge to the default branch so hooks run in agent sessions.

Create a user-level hook

  • Store hook files in ~/.copilot/hooks/ (or $COPILOT_HOME/hooks/) on macOS/Linux.
  • Store hook files in %USERPROFILE%\.copilot\hooks\ (or %COPILOT_HOME%\hooks\) on Windows.
  • Restart Copilot CLI after changes so configurations reload.

Troubleshooting and debugging

  • If hooks do not execute, verify file location, valid JSON, and "version": 1.
  • If hooks time out, raise timeoutSec or optimize scripts.
  • Keep hook JSON output compact and valid (single line).
  • Add script logging and test hooks locally with piped sample input.

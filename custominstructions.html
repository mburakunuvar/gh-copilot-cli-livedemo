Custom Instructions for GitHub Copilot CLI

Define persistent guidance so Copilot CLI automatically uses your project standards, tooling, and workflow details without repeating them in every prompt.

Supported instruction types

  • Repository-wide: .github/copilot-instructions.md applies across the repo.
  • Path-specific: .github/instructions/NAME.instructions.md applies by applyTo glob patterns.
  • Agent instructions: AGENTS.md (plus optional CLAUDE.md/GEMINI.md at repo root).
  • Local: $HOME/.copilot/copilot-instructions.md for your machine only.

Create repository-wide instructions

  1. Create .github/copilot-instructions.md in the repository root.
  2. Write clear natural-language instructions in Markdown.
  3. Save the file; Copilot starts using it automatically.

Create path-specific instructions

  1. Create .github/instructions (and optional subfolders).
  2. Add one or more files ending in .instructions.md.
  3. Add frontmatter with applyTo, e.g. **/*.ts,**/*.tsx.
  4. Optionally set excludeAgent to code-review or cloud-agent.

Behavior notes

  • Changes are available on your next prompt in the current or future CLI sessions.
  • Repo-wide and matching path-specific instructions are combined.
  • Avoid conflicts between instruction files; precedence can be non-deterministic.

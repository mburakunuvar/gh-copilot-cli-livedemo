Adding MCP Servers for GitHub Copilot CLI

MCP lets Copilot CLI connect to external tools and services. The GitHub MCP server is already built in, so you only need to add other servers.

Quick Add in CLI

  1. In interactive mode, run /mcp add.
  2. Set a unique server name and choose a server type: STDIO/Local or HTTP/SSE.
  3. Provide command + optional env vars (STDIO) or URL + optional headers (HTTP/SSE).
  4. Set tools to * (all) or a comma-separated list.
  5. Press Ctrl+S to save. The server is available immediately.

Configure by File

You can also edit ~/.copilot/mcp-config.json to add one or more servers. Define servers under mcpServers with properties such as type, command/args or url/headers, and tools.

Manage Servers

  • /mcp show to list servers and status
  • /mcp show SERVER-NAME for details and tools
  • /mcp edit SERVER-NAME to update configuration
  • /mcp disable SERVER-NAME or /mcp enable SERVER-NAME
  • /mcp delete SERVER-NAME to remove a server

Important Notes

  • If your org/enterprise enforces a registry URL or allowlist, those policies also apply in Copilot CLI.
  • Copilot can automatically use MCP tools when relevant, or you can reference a server/tool directly in your prompt.

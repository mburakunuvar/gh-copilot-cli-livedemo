Adding Agent Skills for GitHub Copilot CLI

Skills are reusable instruction packages (and optional scripts/resources) that Copilot loads when a task matches the skill description.

Create and add a skill

  1. Create a skills root folder:
    • Project: .github/skills, .claude/skills, or .agents/skills
    • Personal: ~/.copilot/skills or ~/.agents/skills
  2. Create one lowercase, hyphenated folder per skill (for example webapp-testing).
  3. Add SKILL.md (required filename) with YAML frontmatter:
    • name (required, unique, lowercase-hyphenated)
    • description (required, when Copilot should use it)
    • license (optional)
  4. Add optional supporting files (scripts/examples/docs) in the same skill folder.

Enable scripts safely

Copilot can access files in a skill directory, so SKILL.md can instruct it to run included scripts.

Optionally add allowed-tools in frontmatter to pre-approve tools like shell or bash.

Security note: only pre-approve terminal tools for trusted skills and reviewed scripts. Otherwise, leave them out so Copilot asks before command execution.

Add a skill from others

  1. Download a skill directory (for example from Awesome GitHub Copilot skills).
  2. Unzip if needed and move it into a valid project or personal skills location.
  3. Start a CLI session, or run /skills reload in an active session.
  4. Verify with /skills info SKILL-NAME.

Use skills in Copilot CLI

Copilot chooses skills automatically from prompt context and each skill's description. You can force one by name, e.g. /frontend-design in your prompt.

  • /skills list — list available skills
  • /skills — enable/disable skills interactively
  • /skills info — show details and location
  • /skills add — add a skills location
  • /skills reload — reload without restarting CLI
  • /skills remove SKILL-DIRECTORY — remove directly added skills

Skills vs custom instructions

Use custom instructions for broad, always-relevant guidance (like coding standards). Use skills for detailed, task-specific procedures that should load only when needed.

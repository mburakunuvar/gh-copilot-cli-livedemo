Creating and Using Custom Agents for GitHub Copilot CLI

Custom agents let Copilot delegate specialized tasks to subagents with separate context windows, keeping your main session focused.

What custom agents are

A custom agent is defined in a Markdown profile file ending in .agent.md. Copilot can call it when your task matches that agent’s expertise.

How to create a custom agent

  1. In interactive mode, run /agent and select Create new agent.
  2. Choose where to store it:
    • Project: .github/agents/
    • User: ~/.copilot/agents/
  3. Pick one creation path:
    • Use Copilot to draft the profile from your description.
    • Create manually by entering name, description, and behavior instructions.
  4. Select tool access (all tools by default, or restricted via a tools specification).
  5. Restart the CLI so the new agent is loaded.

If an agent with the same name exists in both locations, the user-level agent in ~/.copilot/agents/ takes precedence.

How to use a custom agent

  • Slash command: /agent, then pick an agent and enter a prompt.
  • Explicit instruction: ask Copilot to use a named agent in your prompt.
  • By inference: use wording or trigger phrases described in the agent profile.
  • Programmatically: call the CLI with --agent and --prompt.
copilot --agent security-auditor --prompt "Check /src/app/validator.go"

The --agent value is the agent profile filename without the .agent.md suffix.
